# Инструкция по созданию сущности категории "Локации" (Locations)

Каждая локация имеет свой собственный каталог, название которого соответствует названию локации (на английском или транслите без пробелов).
В каталоге создается файл `LoreObject.md`, который содержит актуальное состояние локации на текущий момент.
Если в главе произошли изменения в локации (разрушения, появление новых объектов), создается файл `<Номер главы>.md` с состоянием на момент окончания главы (например, `1.md`).

## Жесткая структура файла LoreObject.md (и файлов истории <Номер главы>.md):

```markdown
# [Название локации]

**Регион:** [Город или более крупный регион]

## Описание
[Общее описание локации, ее назначение, кто ее населяет.]

## Текущее состояние
[Описание текущего состояния, уровень повреждений, активные события в локации.]

## Ключевые объекты
- [Объект 1]
- [Объект 2]
```
